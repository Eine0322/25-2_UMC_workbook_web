- `pushState`, `popstate` 이벤트, 전체 리로드와의 차이 🍠
  pushState(), replaceState() 이벤트
  - 브라우저가 서버로 페이지를 재요청하지 않고 URL만 바꿔준다. 자바스크립트로 해당 URL에 맞게 페이지를 부분 업데이트하기 때문에 빠르다는 장점이 있다.
  전체 리로드
  - 매 로드마다 서버에 페이지를 요청하여 로드한다. 로딩페이지나 깜빡거리는 등 로딩시간이 소요된다.
- 전체 리로드 방식과 SPA 라우팅 방식의 가장 큰 차이는 무엇일까? 🍠
  전체 리로드
  - 새로운 페이지 요청 시 서버에 새로운 html 문서를 요청하고, 페이지 전체를 새로 로드한다. 즉, 페이지 이동마다 새로운 html, css, js 파일 등 모든 리소스를 다시 다운로드 한다. 이로인해 페이지 이동 시 화면이 깜빡거리고 로딩 시간이 발생하여 느리게 느껴질 수 있다.
  SPA 라우팅 방식
  - 페이지 이동 시 서버에 데이터만 요청하고, js 를 이용하여 동적으로 갱신한다. 최초 로드 시에 모든 리소스를 한 번에 다운로드하고, 이후 페이지 로드 시 필요한 데이터만 주고받는다. 이로인해 페이지 이동이 매우 빠르고 부드러워진다.
- `preventDefault()`와 `stopPropagation()`의 차이와 역할은 무엇인가? 🍠
  preventDefault()
  - 이벤트가 발생했을 때, 해당 이벤트에 연결된 브라우저의 기본 동작을 중단시킨다. form 태그의 기본 동작인 submit을 막는 등처럼 사용된다.
  stopPropagation()
  - 중첩 구조에서 자식 요소의 이벤트가 사용한다. 부모 요소의 이벤트 핸들링이 실행되는 것을 막고 싶을 때 사용한다. 특정 이벤트가 다른 요소로 퍼져나가는 것을 막는다.
- 선언적 라우팅(`Route`, `Routes`) 구조가 가지는 장점은 무엇일까? 🍠
  - 가독성, 직관성
    개발자는 해당 컴포넌트가 어떤 URL에 연결되어 있고, 어떤 컴포넌트를 렌더링하는지 한 눈에 파악할 수 있다.
  - 유지보수 및 재사용 용이
    라우팅 로직을 컴포넌트로 분리하여 재사용할 수 있다. JS로직을 사용하면 조건부로 라우트를 추가하는 등의 동적 라우팅이 가능하다.
